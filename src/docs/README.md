# 📚 Документация по интеграции Google Sheets

## Добро пожаловать!

Эта документация поможет вам настроить **автоматическую синхронизацию** между вашим приложением для отслеживания сушки древесины и Google Таблицами.

---

## 🎯 Что это дает?

✅ **Автоматическое закрытие циклов** - старые циклы завершаются автоматически  
✅ **Автоматическое создание новых циклов** - новый цикл создаётся для той же камеры  
✅ **Никакого ручного ввода** - операторы только добавляют строку в Google Sheets  
✅ **Поддержка литовских названий** - используйте родной язык операторов  
✅ **Все данные сохраняются** - фото, оценки, комментарии остаются нетронутыми  
✅ **Автоматическое логирование** - все операции записываются  

---

## 🔄 Как это работает?

1. **Оператор завершает сушку** → Добавляет строку в Google Sheets
2. **Apps Script обнаруживает** → Отправляет данные на сервер
3. **Сервер закрывает старый цикл** → Помечает как завершённый
4. **Ожидание 2-3 минуты** → Даёт время обновиться таблице
5. **Сервер создаёт новый цикл** → С конвертацией литовского названия
6. **Готово!** → Оператор видит новый цикл в приложении

**Одна строка = Завершение старого + Создание нового цикла**

## ⚡ Супер-быстрый старт

### Шаг 1: Создайте Google Таблицу
Добавьте заголовки:
```
Камера | Старый № | Новый № | Порода | Конец старого | Начало нового
```

### Шаг 2: Настройте Apps Script (5 минут)
1. **Расширения** → **Apps Script**
2. Скопируйте код из `google-apps-script.gs`
3. Измените константы (URL, TOKEN, SHEET_NAME)
4. Сохраните

### Шаг 3: Тестируйте (2 минуты)
1. Запустите функцию `testConfiguration()`
2. Добавьте тестовую строку:
```
5 | 0001 | 0002 | Beržas235 | 2024-02-23 10:00:00 | 2024-02-23 10:05:00
```
3. Запустите `manualSync()`
4. Проверьте приложение - новый цикл создан!

### Шаг 4: Автоматизируйте (1 минута)
Запустите `setupTrigger()` или `setupTimeTrigger()`

### Готово! ✅
Теперь при добавлении строк:
- Старый цикл закрывается автоматически
- Новый цикл создаётся через 2-3 минуты
- Операторы работают без ручного ввода!

## 🆘 Частые вопросы

### ❓ Можно ли загружать фото через Google Sheets?
**Нет.** Фотографии загружаются только через приложение (с телефона или компьютера). В таблице хранятся только пути к файлам.

### ❓ Что делать с полем `id`?
**Для новых циклов:** оставьте пустым, оно заполнится автоматически.  
**Для существующих:** никогда не меняйте!

### ❓ Как часто происходит синхронизация?
**Автоматически:** каждые 5 минут.  
**Вручную:** нажмите кнопку "Синхронизировать" в интерфейсе администратора.

### ❓ Что если изменить одну запись в двух местах?
Система выберет последнее изменение (по времени). Рекомендуем избегать одновременного редактирования.

### ❓ Можно ли импортировать старые данные?
Да! См. раздел "Миграция старых данных" в `example-data-ru.md`.

### ❓ Какие данные НЕ нужно заполнять вручную?
- `id` (генерируется автоматически)
- `createdAt` (устанавливается при создании)
- `recipePhotoUrl` (генерируется при загрузке фото)
- Погодные данные (получаются из Open-Meteo API)

### ❓ Нужно ли знать JSON для работы с фотографиями?
Нет! Просто используйте `[]` для пустого массива. При загрузке фото через приложение, JSON обновится автоматически.

## 🎯 Типы пользователей и их задачи

### Администратор:
- Настраивает интеграцию (один раз)
- Создает новые циклы в Google Sheets
- Запускает синхронизацию
- Анализирует данные в таблице

### Оператор:
- Видит новые циклы в приложении (созданные в таблице)
- Загружает фото рецептов
- Работает только с интерфейсом приложения

### Лидер:
- Завершает циклы в приложении
- Загружает фото результатов
- Ставит оценки качества
- Видит обновления в таблице после синхронизации
