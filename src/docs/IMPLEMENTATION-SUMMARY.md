# 📝 Итоги реализации Google Sheets интеграции

## Что было реализовано

Создана полная автоматическая система синхронизации между Google Sheets и приложением для отслеживания циклов сушки древесины.

---

## 🎯 Ключевые особенности

### 1. Событийная модель
- **Одна строка в Google Sheets** = завершение старого цикла + создание нового
- Автоматическое закрытие цикла по 4-значному номеру (`sequentialNumber`)
- Создание нового цикла через 2-3 минуты (задержка для обновления листа)

### 2. Поддержка литовского языка
- Операторы используют литовские названия пород в Google Sheets
- Автоматическая конвертация в английский формат приложения
- Поддержка с/без диакритических знаков (Beržas235 / Berzas235)

### 3. Сохранение данных
- При закрытии старого цикла **НЕ трогаются**:
  - Фотографии рецепта (загруженные оператором)
  - Фотографии результата (загруженные лидером)
  - Конечная влажность
  - Оценка качества
  - Комментарии
- Обновляются только `status` → "Completed" и `endDate`

### 4. Автоматическое логирование
- Все операции записываются с деталями
- Доступны через API endpoint `/sheets/sync-logs`
- Предотвращение дублирования (отслеживание обработанных строк)

## 🌐 API Endpoints

### 1. POST /sheets/process-row
**Назначение:** Webhook для обработки новой строки из Google Sheets

**Тело запроса:**
```json
{
  "rowNumber": 123,
  "chamberNumber": 5,
  "oldSequentialNumber": "0123",
  "newSequentialNumber": "0124",
  "woodTypeLithuanian": "Beržas235",
  "oldCycleEndDate": "2024-02-23T16:00:00Z",
  "newCycleStartDate": "2024-02-23T16:05:00Z"
}
```

**Ответ:**
```json
{
  "success": true,
  "message": "Обработка строки запущена",
  "rowNumber": 123,
  "estimatedTime": "2-3 минуты"
}
```

### 2. GET /sheets/sync-logs?limit=10
**Назначение:** Получение последних логов синхронизации

**Ответ:**
```json
{
  "logs": [
    {
      "timestamp": "2024-02-23T16:00:00Z",
      "logs": [
        {
          "action": "complete_old",
          "success": true,
          "message": "Цикл 0123 завершён..."
        },
        {
          "action": "create_new",
          "success": true,
          "message": "Новый цикл 0124 создан..."
        }
      ]
    }
  ],
  "count": 1
}
```

### 3. GET /sheets/row-status/:rowNumber
**Назначение:** Проверка обработки строки

**Ответ:**
```json
{
  "processed": true,
  "details": {
    "processedAt": "2024-02-23T16:00:00Z",
    "details": { ... }
  }
}
```

### 4. POST /sheets/manual-sync
**Назначение:** Ручная синхронизация (для тестирования)

**Тело запроса:** То же что и для `/sheets/process-row`

**Ответ:**
```json
{
  "success": true,
  "message": "Синхронизация завершена",
  "rowNumber": 123
}
```

## 📊 Процесс обработки

### Временная шкала:

```
T+0 сек:    Google Sheets - новая строка добавлена
T+0-5 сек:  Apps Script обнаруживает изменение
T+5-10 сек: Apps Script отправляет POST запрос на сервер
T+10 сек:   Сервер закрывает старый цикл (status = Completed)
T+10-150 сек: Задержка (2.5 минуты)
T+150 сек:  Сервер создаёт новый цикл
T+150+ сек: Оператор видит новый цикл в приложении
```

### Логика обработки:

1. **Поиск старого цикла:**
   - По `sequentialNumber` == oldSequentialNumber
   - И `chamberNumber` == указанной камеры

2. **Обновление старого:**
   - `status` → "Completed"
   - `endDate` → значение из колонки E
   - Остальные поля НЕ трогаем!

3. **Задержка 2.5 минуты:**
   - `await new Promise(resolve => setTimeout(resolve, 150000))`

4. **Создание нового:**
   - Конвертация породы: литовский → английский
   - Генерация UUID
   - Формирование `recipeCode`
   - Установка `status` = "In Progress"
   - Все остальные поля пустые

## 🔐 Безопасность

### Уровни защиты:

1. **API авторизация:**
   - Все запросы требуют `Authorization: Bearer [ANON_KEY]`
   - ANON_KEY безопасен для использования в клиентских скриптах

2. **Предотвращение дублирования:**
   - Каждая строка обрабатывается только один раз
   - Отслеживание через `processed_row_{rowNumber}` в KV store

3. **Валидация данных:**
   - Проверка обязательных полей
   - Валидация номера камеры (1-21)
   - Валидация типа древесины (из списка)

4. **Логирование:**
   - Все операции записываются с деталями
   - Ошибки логируются с полным контекстом

## 🚀 Быстрый старт (кратко)

1. **Создайте Google Таблицу** с 6 столбцами
2. **Откройте Apps Script** (Расширения → Apps Script)
3. **Вставьте код** из `google-apps-script.gs`
4. **Измените 3 константы:** URL_WEBHOOK, AUTH_TOKEN, SHEET_NAME
5. **Протестируйте:** `testConfiguration()` и `manualSync()`
6. **Автоматизируйте:** `setupTrigger()` или `setupTimeTrigger()`
7. **Готово!** Добавляйте строки и всё работает автоматически

**Время настройки: 15 минут**  
**Экономия времени: часы каждую неделю**
