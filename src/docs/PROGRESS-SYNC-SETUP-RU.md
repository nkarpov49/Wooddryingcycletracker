## 🔧 Настройка Google Sheets

### Шаг 1: Подготовка таблицы

**Ячейки с процентами для каждой камеры:**

```
Камера 1:  C9
Камера 2:  E9
Камера 3:  G9
Камера 4:  I9
Камера 5:  K9
Камера 6:  M9
Камера 7:  O9
Камера 8:  Q9
Камера 9:  S9
Камера 10: U9
Камера 11: W9
Камера 12: Y9
Камера 13: AA9
Камера 14: AC9
Камера 15: AE9
Камера 16: AG9
Камера 17: AI9
Камера 18: AK9
Камера 19: AM9
Камера 20: AO9
Камера 21: AQ9
```

**Формула процента (уже установлена):**
```
=(now()-$C$6)/((vlookup($C$4;Standartai!$B$5:$E;4;false))/24)
```

**Важно:**
- Формула возвращает **дробное число** (0.59 = 59%)
- Скрипт **автоматически умножает на 100**
- Проценты могут быть **>100** (например: 115.5%)

### Шаг 2: Создание Apps Script

1. Откройте вашу Google Sheets таблицу
2. Перейдите в: **Расширения → Apps Script**
3. Создайте новый файл и скопируйте код из `/docs/google-apps-script-progress.txt`
4. **Настройте параметры:**

```javascript
// Ваш URL Supabase проекта
const SUPABASE_URL = 'https://ваш-project-id.supabase.co';

// Диапазон с процентами (столбец с формулами для камер 1-21)
// Пример: если проценты в колонке G строках 2-22
const PROGRESS_RANGE = 'G2:G22';
```

### Шаг 3: Тестирование

1. В Apps Script нажмите на функцию `testSyncProgress()`
2. Нажмите **Выполнить** (▶️)
3. Разрешите доступ (первый раз)
4. Посмотрите логи (Ctrl+Enter или View → Logs)
5. Проверьте что данные пришли на сервер

**Ожидаемый результат в логах:**
```
✅ Синхронизация успешна: Обновлено 21 из 21 камер
```

### Шаг 4: Настройка автоматического триггера

1. В Apps Script нажмите на значок **часов** ⏰ (Триггеры)
2. Нажмите **"+ Добавить триггер"**
3. Настройте параметры:
   - **Функция:** `syncProgressToApp`
   - **Источник события:** `Временной`
   - **Тип таймера:** `Таймер с интервалом`
   - **Интервал:** `Каждые 15 минут`
4. Сохраните

**Готово!** Теперь проценты будут автоматически синхронизироваться каждые 15 минут.

    if (c.progressPercent === undefined) {
      // Разные сценарии для демонстрации:

      if (chamberNum % 7 === 0) {
        // Каждая 7-я камера в критической зоне (99-103%)

    }

setCycles(enrichedData.filter((c: any) => c.status === 'In Progress' && !c.endDate));

## 🧪 Тестирование функционала

### 1. Проверка отображения процента

1. Создайте несколько тестовых циклов
2. Запустите Google Apps Script (вручную или дождитесь триггера)
3. Обновите страницу оператора
4. Проценты должны отображаться в карточках камер

### 2. Проверка критической зоны (99-103%)

1. Установите процент для камеры в диапазоне 99-103%
2. НЕ загружайте фото рецепта
3. Карточка должна **мигать красным**
4. **Через 2 минуты** должно появиться уведомление и звук
5. Уведомления повторяются каждые 2 минуты

### 3. Проверка после загрузки фото

1. Сфотографируйте рецепт для критической камеры
2. Мигание должно **прекратиться**
3. Уведомления больше **не приходят**
4. Статус меняется на "Фото загружено"

## 📝 Формат данных API

### Запрос (POST `/sheets/sync-progress`)

```json
{
  "1": 45,
  "2": 78,
  "3": 99,
  "4": 15,
  "5": 101,
  ...
  "21": 12
}
```

**Особенности:**
- Ключи: строки "1"-"21" (номера камер)
- Значения: числа 0-100+ (проценты)
- Значения >100 допустимы (некоторые рецепты могут превышать 100%)

### Ответ (успех)

```json
{
  "success": true,
  "message": "Обновлено 18 из 21 камер",
  "updates": [
    { "chamber": 1, "progress": 45, "success": true },
    { "chamber": 2, "progress": 78, "success": true },
    { "chamber": 3, "progress": 0, "success": false, "error": "No active cycle" },
    ...
  ]
}
```

**Примечание:** Камеры без активного цикла не обновляются (это нормально).

## 🐛 Решение проблем

### Проблема: Проценты не обновляются

**Решения:**
1. Проверьте URL в Google Apps Script (`SUPABASE_URL`)
2. Проверьте диапазон ячеек (`PROGRESS_RANGE`)
3. Посмотрите логи Apps Script (View → Logs)
4. Проверьте логи сервера (Supabase Dashboard → Edge Functions → Logs)

### Проблема: Уведомления не работают

**Решения:**
1. Разрешите уведомления в браузере
2. Проверьте что камера в критической зоне (99-103%)
3. Проверьте что фото НЕ загружено
4. Подождите 2 минуты (первое уведомление не сразу)

### Проблема: Звук не воспроизводится

**Решения:**
1. Проверьте что звук браузера не отключен
2. Некоторые браузеры блокируют автовоспроизведение
3. Кликните на страницу перед первым уведомлением

## 📱 Для операторов

### Как работают уведомления:

1. **Когда появляются:**
   - Процент достигает 99-103%
   - Фото рецепта НЕ загружено

   - 🔴 Мигающая красная карточка камеры
   - 🔔 Push-уведомление браузера
   - 🔊 Звуковой сигнал (3 beep-а)

3. **Как отключить:**
   - Сфотографируйте рецепт
   - Уведомления прекратятся автоматически
