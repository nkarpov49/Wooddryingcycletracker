# ⚠️ ЧТО ДЕЛАТЬ СЕЙЧАС: Лимит urlfetch исчерпан

## 🚨 Текущая ситуация

```
❌ Ошибка: Service invoked too many times for one day: urlfetch
❌ Ошибка 401: Missing authorization header
```

**Это означает:**
1. Лимит Google Apps Script на сегодня **ИСЧЕРПАН**
2. В коде **НЕТ заголовка Authorization** (SUPABASE_ANON_KEY)
3. Вы не можете отправлять HTTP-запросы до завтра
4. Это произошло из-за многократных тестовых запусков

## ⏰ ГЛАВНОЕ: Подождите до завтра!

```
Лимиты сбрасываются каждые 24 часа
По времени Pacific Time (США)

🕐 Для Литвы (Вильнюс):
   Сброс около 21:00-22:00 вечера

🕐 Для России (Москва):
   Сброс около 02:00-03:00 ночи
```

## ✅ ЧТО СДЕЛАТЬ ПРЯМО СЕЙЧАС (5 минут)

### 1. Удалите ВСЕ триггеры
```
Apps Script → Triggers (иконка часов) → Delete All
```
**Зачем?** Чтобы не было дублирующихся триггеров завтра

### 2. НЕ запускайте скрипт вручную
```
НЕ нажимайте кнопку Run (▶️) больше сегодня!
```
**Зачем?** Чтобы не ухудшить ситуацию

### 3. Проверьте что код обновлён
```
Откройте Apps Script → Проверьте что есть:

// В начале файла:
const SUPABASE_ANON_KEY = 'ваш-anon-key-здесь';

// В fetch options:
'headers': {
  'apikey': SUPABASE_ANON_KEY  // ← ЭТА СТРОКА ДОЛЖНА БЫТЬ!
}

// И умножение на 100:
if (typeof percentValue === 'number') {
  percentValue = percentValue * 100;  // ← И ЭТА ТОЖЕ!
}
```

### 4. 🔑 ВАЖНО: Добавьте SUPABASE_ANON_KEY
```
⚠️ ОБЯЗАТЕЛЬНО замените в коде:
const SUPABASE_ANON_KEY = 'ваш-anon-key-здесь';

Где найти?
Supabase Dashboard → Project Settings → API → "anon" key

См. детали: /docs/FIX-401-ERROR-RU.md
```

### 5. (Опционально) Запустите тест БЕЗ отправки
```
См. раздел ниже "Тестовый скрипт"
```

## 🧪 ОПЦИОНАЛЬНО: Тестовый скрипт (БЕЗ urlfetch)

Если хотите **проверить** что данные читаются правильно:

### Шаг 1: Создайте новый файл
```
Apps Script → File → New → Script file
Имя: "TestOnly"
```

### Шаг 2: Скопируйте код
```
Откройте: /docs/google-apps-script-TEST-ONLY.txt
Скопируйте ВЕСЬ код
Вставьте в файл "TestOnly"
Сохраните
```

### Шаг 3: Запустите тест
```
Выберите функцию: testProgressReadingOnly
Нажмите Run (▶️)
```

### Шаг 4: Проверьте логи
```
View → Logs (или Ctrl+Enter)

ОЖИДАЕМЫЙ РЕЗУЛЬТАТ:
✅ Чтение завершено успешно!

📊 РЕЗУЛЬТАТЫ:
{
  "1": 59.2,   ← Правильно (НЕ 59!)
  "2": 23.4,   ← Правильно (НЕ 2!)
  "3": 115.5,  ← Правильно (с десятичной)
  ...
}
```

**Этот тест НЕ использует urlfetch!** Он покажет только что данные читаются правильно.

## 🕐 ЗАВТРА УТРОМ (после сброса лимитов)

### Шаг 1: Проверьте что лимиты сброшены
```
Apps Script → Run основной скрипт → Проверьте логи
Если ошибки urlfetch НЕТ - всё ОК!
```

### Шаг 2: Создайте ОДИН триггер
```
Triggers → Add Trigger

Function: syncProgressToApp
Event: Time-driven
Type: Minutes timer
Interval: Every 15 minutes

ВАЖНО: Только ОДИН триггер!
```

### Шаг 3: Проверьте первый запуск
```
Подождите 15 минут
Executions → Проверьте последний запуск
Должно быть: ✅ Синхронизация успешна
```

### Шаг 4: Проверьте интерфейс
```
Откройте Make → Страница "Оператор"
Проверьте проценты на карточках камер
Должны быть: 59.2%, 23.4%, 115.5% (правильные)
```

## ❓ Частые вопросы

### ❓ Могу ли я что-то сделать СЕГОДНЯ?
Нет, лимит urlfetch исчерпан. Можно только запустить тестовый скрипт (без отправки).

### ❓ Точно ли заработает завтра?
Да, если:
1. Удалите все триггеры сегодня
2. Создадите только ОДИН триггер завтра
3. Не будете запускать скрипт вручную много раз

### ❓ Как узнать когда сбросятся лимиты?
Примерно в 21:00-22:00 по Вильнюсу (или 02:00-03:00 по Москве).

### ❓ Что если ошибка повторится завтра?
1. Проверьте что нет дублирующихся триггеров
2. Проверьте Executions (сколько раз запускался скрипт)
3. Увеличьте интервал до 30 минут (вместо 15)

### ❓ Можно ли использовать другой Google аккаунт?
Да, но проще подождать до завтра.
