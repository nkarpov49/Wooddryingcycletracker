## 📝 Проверка НОВОЙ записи

Перед добавлением новой строки в Google Sheets, убедитесь:

### Обязательные поля:
- [ ] **Столбец A (id):** ПУСТОЙ (не заполнен)
- [ ] **Столбец B (chamberNumber):** Число от 1 до 21
- [ ] **Столбец C (sequentialNumber):** Заполнено (например: 2024-001)
- [ ] **Столбец D (recipeCode):** Заполнено (например: БР235-48-30)
- [ ] **Столбец E (woodType):** Один из списка: Birch235, Birch285, Alder235, Oak235, Ash235, Maple235, Scroblas235
- [ ] **Столбец F (status):** `In Progress` (с заглавными буквами и пробелом)
- [ ] **Столбец G (createdAt):** Дата в формате `YYYY-MM-DDTHH:mm:ssZ`
- [ ] **Столбец H (startDate):** Дата в формате `YYYY-MM-DDTHH:mm:ssZ`

### Опциональные поля (можно оставить пустыми):
- [ ] **Столбец I (customWoodType):** Пусто или текст
- [ ] **Столбец J (isBaseRecipe):** `FALSE` или `TRUE` (заглавные буквы)
- [ ] **Столбец K (isTest):** `FALSE` или `TRUE` (заглавные буквы)

### Поля которые должны быть пустыми для новой записи:
- [ ] **Столбец L (endDate):** ПУСТО
- [ ] **Столбец M (finalMoisture):** ПУСТО
- [ ] **Столбец N (qualityRating):** ПУСТО
- [ ] **Столбец O (overallComment):** Пусто (или можно заполнить)

### Поля погоды (оставьте пустыми):
- [ ] **Столбцы P-V:** ВСЕ ПУСТЫЕ

### Поля фотографий:
- [ ] **Столбец W (recipePhotos):** `[]` (пустой массив)
- [ ] **Столбец X (resultPhotos):** `[]` (пустой массив)

### Устаревшие поля (не заполняйте):
- [ ] **Столбец Y (recipePhotoPath):** ПУСТО
- [ ] **Столбец Z (recipePhotoUrl):** ПУСТО

## 📝 Проверка ОБНОВЛЕНИЯ существующей записи

При изменении существующей строки, проверьте:

### Основные поля:
- [ ] **Столбец A (id):** НЕ ИЗМЕНЕН (UUID остался тем же)
- [ ] **Столбец B (chamberNumber):** Если изменили, то число от 1 до 21
- [ ] **Столбец E (woodType):** Если изменили, то один из списка
- [ ] **Столбец F (status):** `In Progress` или `Completed` (с заглавными)

### Если завершаете цикл:
- [ ] **Столбец F (status):** Изменено на `Completed`
- [ ] **Столбец L (endDate):** Добавлена дата в формате ISO
- [ ] **Столбец M (finalMoisture):** Добавлено число (влажность)
- [ ] **Столбец N (qualityRating):** Добавлено число от 0.5 до 5.0
- [ ] **Столбец O (overallComment):** Можно добавить комментарий

### Чего НЕ делать:
- [ ] **НЕ изменяйте id** (столбец A)
- [ ] **НЕ меняйте createdAt** (столбец G)
- [ ] **НЕ заполняйте погоду вручную** (столбцы P-V, если уже заполнены автоматически)
- [ ] **НЕ добавляйте recipePhotoUrl** (столбец Z)

## 📝 Проверка формата ДАТЫ

Проверьте каждую дату в столбцах G, H, L:

- [ ] Формат: `YYYY-MM-DDTHH:mm:ssZ`
- [ ] Год: 4 цифры (например: 2024)
- [ ] Месяц: 2 цифры от 01 до 12
- [ ] День: 2 цифры от 01 до 31
- [ ] Есть разделитель `T` между датой и временем
- [ ] Время: формат `HH:mm:ss` (например: 10:30:00)
- [ ] В конце есть `Z` (означает UTC)

❌ **Неправильно:**
```
23.02.2024
2024-2-23
2024-02-23
2024-02-23 10:30:00
```

✅ **Правильно:**
```
2024-02-23T10:30:00Z
```

## 📝 Проверка ЧИСЛОВЫХ полей

### chamberNumber (столбец B):
- [ ] Целое число
- [ ] От 1 до 21
- [ ] Не пусто

### finalMoisture (столбец M):
- [ ] Число с десятичной точкой (не запятой!)
- [ ] Положительное значение
- [ ] Обычно от 5.0 до 15.0
- [ ] Пример: `8.2` (не `8,2`)

### qualityRating (столбец N):
- [ ] Число от 0.5 до 5.0
- [ ] С шагом 0.5 (0.5, 1.0, 1.5, 2.0, ..., 5.0)
- [ ] Используйте точку, не запятую
- [ ] Примеры: `4.5`, `3.0`, `5.0`

❌ **Неправильно:**
```
4,5    ← запятая вместо точки
6.0    ← больше 5.0
4.3    ← не кратно 0.5
```

✅ **Правильно:**
```
4.5
3.0
5.0
```

## 📝 Проверка JSON (Фотографии)

### recipePhotos (столбец W) и resultPhotos (столбец X):

#### Пустой массив (нет фото):
- [ ] Точно: `[]` (квадратные скобки без пробелов)

#### С фотографиями:
- [ ] Начинается с `[` и заканчивается `]`
- [ ] Используются двойные кавычки `"`
- [ ] Формат: `[{"path":"filename.jpg","caption":"Текст"}]`
- [ ] НЕТ пробелов после запятых (Google Sheets может неправильно распарсить)
- [ ] НЕ добавляйте поле `url` вручную

❌ **Неправильно:**
```
[{"path": "file.jpg"}]              ← пробелы после двоеточия
[{'path':'file.jpg'}]               ← одинарные кавычки
[{"path":"file.jpg","url":"..."}]   ← вручную добавлен url
file.jpg                            ← не JSON
```

✅ **Правильно:**
```
[]
[{"path":"file.jpg"}]
[{"path":"file.jpg","caption":"Фото 1"}]
[{"path":"f1.jpg"},{"path":"f2.jpg"}]
```

## 🔧 Автоматическая проверка в Google Sheets

### Добавьте эти формулы для автоматической валидации:

#### Проверка обязательных полей (новая колонка):
```
=IF(A2="", 
  IF(AND(B2<>"", C2<>"", D2<>"", E2<>"", F2<>"", G2<>"", H2<>""), 
    "✅ Новая запись OK", 
    "❌ Заполните все обязательные поля"
  ),
  "✅ Существующая запись"
)
```

#### Проверка номера камеры:
```
=IF(AND(B2>=1, B2<=21), "✅", "❌ Камера должна быть 1-21")
```

#### Проверка оценки:
```
=IF(OR(N2="", AND(N2>=0.5, N2<=5, MOD(N2*2,1)=0)), "✅", "❌ Оценка 0.5-5.0 с шагом 0.5")
```

#### Проверка статуса:
```
=IF(OR(F2="In Progress", F2="Completed"), "✅", "❌ Статус неверный")
```

#### Проверка типа дерева:
```
=IF(OR(E2="Birch235", E2="Birch285", E2="Alder235", E2="Oak235", E2="Ash235", E2="Maple235", E2="Scroblas235"), "✅", "❌ Тип дерева неверный")
```

## 📊 Условное форматирование

Настройте цветовую подсветку ошибок:

### Красный фон для некорректных данных:

1. **Столбец B (chamberNumber):**
   - Формула: `=OR(B2<1, B2>21, NOT(ISNUMBER(B2)))`
   - Формат: Красный фон

2. **Столбец E (woodType):**
   - Формула: `=AND(E2<>"", NOT(OR(E2="Birch235", E2="Birch285", E2="Alder235", E2="Oak235", E2="Ash235", E2="Maple235", E2="Scroblas235")))`
   - Формат: Красный фон

3. **Столбец F (status):**
   - Формула: `=AND(F2<>"", NOT(OR(F2="In Progress", F2="Completed")))`
   - Формат: Красный фон

4. **Столбец N (qualityRating):**
   - Формула: `=AND(N2<>"", OR(N2<0.5, N2>5, MOD(N2*2,1)<>0))`
   - Формат: Красный фон

### Желтый фон для предупреждений:

1. **Низкая оценка (<3.0):**
   - Формула: `=AND(N2<>"", N2<3)`
   - Формат: Желтый фон

2. **Незавершенный цикл старше 7 дней:**
   - Формула: `=AND(F2="In Progress", H2<>"", NOW()-H2>7)`
   - Формат: Желтый фон
