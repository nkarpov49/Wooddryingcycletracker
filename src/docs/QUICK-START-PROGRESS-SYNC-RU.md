# 🚀 Быстрый старт: Синхронизация процентов из Google Sheets

## ⚠️ ВАЖНО: Если видите ошибку urlfetch

```
Exception: Service invoked too many times for one day: urlfetch
```

**→ Откройте**: `/docs/URGENTLY-NOW-RU.md`  
**Краткое решение**: Подождите до завтра, лимит сбросится автоматически

## ✅ Что исправлено

1. **Правильное чтение процентов**: теперь формула `=(now()-$C$6)/((vlookup(...))` возвращает дробное число (0.59 = 59%), которое автоматически умножается на 100
2. **Поддержка десятичных**: проценты отображаются с точностью до 0.1 (например: **115.5%**)
3. **Трёхзначные проценты**: >100% поддерживаются и отображаются фиолетовым цветом 🟣
4. **Защита от лимитов**: инструкция по решению ошибки urlfetch

## 📋 Быстрая настройка (5 минут)

### Шаг 1: Скопируйте скрипт
```
Откройте файл: /docs/google-apps-script-progress.txt
Скопируйте ВЕСЬ код
```

### Шаг 2: Вставьте в Google Apps Script
```
Google Sheets → Расширения → Apps Script
Вставьте код
```

### Шаг 3: ⚠️ ОБЯЗАТЕЛЬНО замените настройки
```javascript
// Найдите в начале файла:
const SUPABASE_URL = 'https://ваш-project-id.supabase.co';
const SUPABASE_ANON_KEY = 'ваш-anon-key-здесь';

// Замените на ВАШИ значения!
```

**Где найти эти значения?**
1. **SUPABASE_URL**: Ваш URL проекта (например: `https://abc123.supabase.co`)
2. **SUPABASE_ANON_KEY**: 
   - Откройте Supabase Dashboard
   - Project Settings → API
   - Скопируйте "anon" / "public" key

### Шаг 4: Создайте триггер
```
Apps Script → Triggers (иконка часов)
Add Trigger:
  - Function: syncProgressToApp
  - Event: Time-driven
  - Type: Minutes timer
  - Interval: Every 15 minutes
```

### Шаг 5: Тестирование

1. Нажмите **Run** (▶️) в Apps Script
2. Дайте разрешения Google (первый раз)
3. Проверьте логи (View → **Executions**)
4. Должно быть: 
   ```
   ✅ Синхронизация успешна
   Обновлено камер: 21
   ```

## ⚠️ Решение проблемы "urlfetch limit"

Если видите ошибку:
```
Exception: Service invoked too many times for one day: urlfetch
```

**Причина**: Слишком много триггеров или тестовых запусков

**Решение**:
1. Apps Script → **Triggers** (часы)
2. **Удалите ВСЕ** триггеры для `syncProgressToApp`
3. Создайте **ОДИН НОВЫЙ** (каждые 15 минут)
4. Подождите до завтра (лимиты сбрасываются каждые 24 часа)

**Детали**: См. `/docs/GOOGLE-SHEETS-URLFETCH-LIMIT-FIX.md`

## 📊 Расчёт использования

- **Интервал**: каждые 15 минут
- **Запросов в день**: ~96
- **Лимит Google**: 20,000 запросов/день
- **Вывод**: ✅ Укладываемся с большим запасом

Если ошибка повторяется:
1. Проверьте количество триггеров (должен быть **1**)
2. Проверьте **Executions** (список запусков)
3. Увеличьте интервал до 30 минут (опционально)

2. Найдите секцию:
   ```typescript
   // 🎭 MOCK DATA: Добавляем случайные проценты для демонстрации
   ```
3. Удалите весь блок кода с mock-данными
4. Оставьте только:
   ```typescript
   const data = await api.getCycles();
   setCycles(data.filter((c: any) => c.status === 'In Progress' && !c.endDate));
   ```

## 🆘 Частые проблемы

| Проблема | Решение |
|----------|---------|
| Проценты = 0 | Проверьте что формула в правильной ячейке (C9, E9...) |
| Проценты слишком малые (2, 3, 1) | Скопируйте ОБНОВЛЁННЫЙ код из `/docs/google-apps-script-progress.txt` |
| Ошибка urlfetch | Удалите лишние триггеры, оставьте один |
| Нет обновлений | Проверьте SUPABASE_URL в коде Apps Script |
| 403/401 ошибка | Проверьте что endpoint `/sheets/sync-progress` доступен |
