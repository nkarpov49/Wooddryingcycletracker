# 🔄 Реальный рабочий процесс синхронизации

## Описание процесса

Система работает на основе **событийной модели**: когда оператор завершает сушку, в Google Sheets появляется новая строка, которая автоматически:
1. **Закрывает старый цикл** (помечает как завершённый)
2. **Создаёт новый цикл** для той же камеры (через 2-3 минуты)

**Важно:** Одна строка = одно событие завершения + одно создание

---

## 📋 Структура строки в Google Sheets

| Столбец | Поле | Что содержит | Пример |
|---------|------|--------------|--------|
| A | Номер камеры | Камера сушки (1-21) | `5` |
| B | Старый номер | 4-значный номер завершающегося цикла | `0123` |
| C | Новый номер | 4-значный номер нового цикла | `0124` |
| D | Порода | **Литовское** название породы | `Beržas235` |
| E | Конец старого | Дата/время завершения старого цикла | `2024-02-23 16:00:00` |
| F | Начало нового | Дата/время начала нового цикла | `2024-02-23 16:05:00` |

### Пример строки:
```
5    0123    0124    Beržas235    2024-02-23 16:00:00    2024-02-23 16:05:00
```

Эта строка означает:
- Камера №5
- Цикл №0123 завершён 23.02.2024 в 16:00
- Цикл №0124 начался 23.02.2024 в 16:05
- Порода: Береза 235

## 📖 Маппинг литовских названий пород

Система автоматически конвертирует литовские названия в английский формат:

| Литовский | Английский | Русский |
|-----------|------------|---------|
| `Beržas235` или `Berzas235` | `Birch235` | Берёза 235 |
| `Beržas285` или `Berzas285` | `Birch285` | Берёза 285 |
| `Alksnis235` | `Alder235` | Ольха 235 |
| `Ąžuolas235` или `Azuolas235` | `Oak235` | Дуб 235 |
| `Uosis235` | `Ash235` | Ясень 235 |
| `Klevas235` | `Maple235` | Клён 235 |
| `Skroblas235` | `Scroblas235` | Скробла 235 |

**Важно:** В Google Sheets используйте **литовские** названия, система автоматически переведёт их!

## 🎯 Что заполняется автоматически

### При закрытии старого цикла:
- ✅ `status` → `"Completed"`
- ✅ `endDate` → время из колонки E

### Что НЕ меняется при закрытии:
- ❌ Фотографии рецепта (загружались ранее оператором)
- ❌ Фотографии результата (загружались лидером)
- ❌ Конечная влажность (вводилась лидером)
- ❌ Оценка качества (вводилась лидером)
- ❌ Комментарий (вводился лидером)

### При создании нового цикла:
- ✅ `id` → новый UUID
- ✅ `chamberNumber` → из колонки A
- ✅ `sequentialNumber` → из колонки C (новый номер)
- ✅ `woodType` → конвертируется из колонки D
- ✅ `recipeCode` → генерируется автоматически
- ✅ `startDate` → время из колонки F
- ✅ `status` → `"In Progress"`
- ✅ `createdAt` → время создания

### Что пустое в новом цикле:
- ⚪ `recipePhotos` → `[]` (оператор загрузит позже)
- ⚪ `resultPhotos` → `[]` (лидер загрузит при завершении)
- ⚪ `finalMoisture` → `null`
- ⚪ `qualityRating` → `null`
- ⚪ `overallComment` → `""`

## 🛠 Настройка Google Apps Script

### Шаг 1: Создание скрипта

1. Откройте вашу Google Таблицу
2. Меню: **Расширения** → **Apps Script**
3. Скопируйте код из файла `/docs/google-apps-script.gs`
4. Вставьте в редактор

### Шаг 2: Настройка констант

В начале скрипта измените:

```javascript
// URL вашего Supabase Edge Function
const URL_WEBHOOK = 'https://[ваш-project-id].supabase.co/functions/v1/make-server-c5bcdb1f/sheets/process-row';

// Ваш SUPABASE_ANON_KEY
const AUTH_TOKEN = 'ваш_anon_key_здесь';

// Название вашего листа
const SHEET_NAME = 'Лист1';
```

### Шаг 3: Настройка столбцов (если отличаются)

Если ваши данные в других столбцах, измените:

```javascript
const COLUMNS = {
  CHAMBER_NUMBER: 1,        // Столбец A
  OLD_SEQUENTIAL: 2,        // Столбец B
  NEW_SEQUENTIAL: 3,        // Столбец C
  WOOD_TYPE_LITHUANIAN: 4,  // Столбец D
  OLD_END_DATE: 5,          // Столбец E
  NEW_START_DATE: 6         // Столбец F
};
```

### Шаг 4: Тестирование

Запустите функцию `testConfiguration()`:
1. Выберите функцию в dropdown
2. Нажмите "Выполнить"
3. Проверьте логи: **Вид** → **Журналы**

### Шаг 5: Ручная синхронизация

Проверьте работу на последней строке:
1. Запустите функцию `manualSync()`
2. Проверьте логи
3. Убедитесь, что старый цикл закрыт и новый создан

### Шаг 6: Автоматизация

Выберите один вариант:

**Вариант A: Автоматически при изменении**
```javascript
setupTrigger()  // Запустите эту функцию
```

**Вариант B: Каждые 5 минут**
```javascript
setupTimeTrigger()  // Запустите эту функцию
```

## 📊 Пример использования

### Исходная ситуация:
- Камера №5 работает с циклом №0123
- Оператор загрузил фото рецепта через приложение
- Лидер добавил влажность, оценку, фото результата
- Сушка завершена

### Действие оператора:
Оператор добавляет строку в Google Sheets:
```
5    0123    0124    Beržas235    2024-02-23 16:00:00    2024-02-23 16:05:00
```

### Что происходит автоматически:

**Через ~5 секунд:**
- Цикл №0123:
  - `status` → `"Completed"`
  - `endDate` → `"2024-02-23T16:00:00Z"`
  - Все данные (фото, оценка, влажность) сохранены
  - Карточка помечена как завершённая

**Через ~2.5 минуты:**
- Цикл №0124 создан:
  - `chamberNumber` → `5`
  - `sequentialNumber` → `"0124"`
  - `woodType` → `"Birch235"` (конвертировано из Beržas235)
  - `status` → `"In Progress"`
  - `startDate` → `"2024-02-23T16:05:00Z"`
  - Все поля пустые (готовы для заполнения)

### Результат:
- ✅ Старый цикл завершён и доступен в истории
- ✅ Новый цикл активен и виден оператору
- ✅ Оператор может сфотографировать рецепт для нового цикла

## ⚠️ Важные моменты

### 1. Не создавайте циклы вручн��ю в приложении
Все циклы создаются автоматически через Google Sheets. Приложение используется только для:
- Загрузки фото рецепта (оператор)
- Завершения цикла с фото результата (лидер)
- Просмотра истории (все роли)

### 2. Используйте литовские названия пород
В Google Sheets всегда используйте литовские названия:
- ✅ `Beržas235`
- ✅ `Klevas235`
- ❌ `Birch235` (не используйте английские)

### 3. Задержка 2-3 минуты обязательна
Система ждёт, чтобы Google Sheets обновил номера циклов. Не пытайтесь ускорить этот процесс.

### 4. Четырёхзначные номера
Номера циклов должны быть 4-значными:
- ✅ `0123`
- ✅ `0001`
- ❌ `123` (недостаточно нулей)

### 5. Уникальность строк
Система отслеживает обработанные строки по номеру строки. Каждая строка обрабатывается только один раз.

## 🆘 Решение проблем

### Проблема: Старый цикл не закрывается
**Причины:**
- Номер цикла в колонке B не совпадает с `sequentialNumber` в базе
- Номер камеры не совпадает

**Решение:**
- Проверьте точное совпадение номера (включая нули)
- Проверьте номер камеры

### Проблема: Новый цикл не создаётся
**Причины:**
- Литовское название породы не распознано
- Ошибка конвертации

**Решение:**
- Используйте точные названия из таблицы маппинга
- Проверьте логи сервера: `GET /sheets/sync-logs`

### Проблема: Скрипт не срабатывает автоматически
**Причины:**
- Триггер не настроен
- Триггер отключён

**Решение:**
- Запустите `setupTrigger()` или `setupTimeTrigger()`
- Проверьте триггеры: **Триггеры** → **Управление триггерами проекта**

### Проблема: Дублирование циклов
**Причины:**
- Строка обработана несколько раз

**Решение:**
- Система автоматически предотвращает дублирование
- Проверьте статус строки: `GET /sheets/row-status/:rowNumber`
