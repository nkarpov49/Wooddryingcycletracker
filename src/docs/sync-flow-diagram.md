# Диаграмма процесса синхронизации

## Общая схема работы

```
┌─────────────────────────────────────────────────────────────────────┐
│                         СИНХРОНИЗАЦИЯ ДАННЫХ                         │
└─────────────────────────────────────────────────────────────────────┘

┌──────────────────────┐                    ┌──────────────────────┐
│                      │   ← Двусторонняя   │                      │
│   Google Sheets      │      синхронизация │  Веб-приложение      │
│   (Ваша таблица)     │   →                │  (Supabase KV)       │
│                      │                    │                      │
└──────────────────────┘                    └──────────────────────┘
         ↓                                            ↓
    21 камера                                   21 камера
  Все циклы сушки                            Все циклы сушки
```

## Сценарий 1: Создание нового цикла В ПРИЛОЖЕНИИ

```
┌─────────────────────────────────────────────────────────────────────┐
│ ОПЕРАТОР: Создает новый цикл через интерфейс                        │
└─────────────────────────────────────────────────────────────────────┘
                               ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 1. Оператор заполняет форму:                                        │
│    • Камера №5                                                      │
│    • Порядковый номер: 2024-045                                     │
│    • Код рецепта: БР235-48-30                                       │
│    • Тип дерева: Birch235                                           │
│    • Фото рецепта (загружает с телефона)                            │
└─────────────────────────────────────────────────────────────────────┘
                               ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 2. Приложение сохраняет данные:                                     │
│    • Генерирует UUID                                                │
│    • Загружает фото в Supabase Storage                              │
│    • Сохраняет цикл в Supabase KV                                   │
│    • Устанавливает status = "In Progress"                           │
│    • Добавляет startDate = текущее время                            │
└─────────────────────────────────────────────────────────────────────┘
                               ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 3. АВТОМАТИЧЕСКАЯ СИНХРОНИЗАЦИЯ (каждые 5 мин):                     │
│    • Сервер проверяет новые циклы в KV                              │
│    • Находит цикл без отметки "synced_to_sheets"                    │
│    • Добавляет новую строку в Google Sheets                         │
│    • Помечает цикл как синхронизированный                           │
└─────────────────────────────────────────────────────────────────────┘
                               ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 4. Результат:                                                       │
│    ✅ Цикл виден в приложении                                       │
│    ✅ Цикл виден в Google Таблице                                   │
│    ✅ Оператор может продолжать работу                              │
└─────────────────────────────────────────────────────────────────────┘
```

## Сценарий 3: Завершение цикла В ПРИЛОЖЕНИИ

```
┌─────────────────────────────────────────────────────────────────────┐
│ ЛИДЕР: Завершает цикл через интерфейс                               │
└─────────────────────────────────────────────────────────────────────┘
                               ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 1. Лидер открывает существующий цикл:                               │
│    • Добавляет finalMoisture: 8.2%                                  │
│    • Загружает фото результата                                      │
│    • Ставит оценку qualityRating: 4.5 звезд                         │
│    • Добавляет комментарий: "Отличное качество"                     │
└─────────────────────────────────────────────────────────────────────┘
                               ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 2. Приложение обновляет цикл:                                       │
│    • Загружает фото в Supabase Storage                              │
│    • Автоматически устанавливает status = "Completed"               │
│    •   (т.к. есть finalMoisture + rating + resultPhoto)             │
│    • Устанавливает endDate = текущее время                          │
│    • Сохраняет в Supabase KV                                        │
└─────────────────────────────────────────────────────────────────────┘
                               ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 3. АВТОМАТИЧЕСКАЯ СИНХРОНИЗАЦИЯ:                                    │
│    • Сервер находит обновленный цикл                                │
│    • Ищет строку в Google Sheets по id                              │
│    • Обновляет все поля (status, endDate, rating, etc.)             │
│    • Обновляет JSON с путями к фото результата                      │
└─────────────────────────────────────────────────────────────────────┘
                               ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 4. Результат:                                                       │
│    ✅ Статус изменился на "Completed" в обоих местах                │
│    ✅ Все данные синхронизированы                                   │
│    ✅ В таблице видны все результаты                                │
└─────────────────────────────────────────────────────────────────────┘
```

## Обработка конфликтов

```
┌─────────────────────────────────────────────────────────────────────┐
│ КОНФЛИКТ: Одна запись изменена в двух местах одновременно           │
└─────────────────────────────────────────────────────────────────────┘

Google Sheets:                          Приложение:
status = "Completed"                    overallComment = "Новый комментарий"
updatedAt = 10:05:00                    updatedAt = 10:06:00

                               ↓
                      СИНХРОНИЗАЦИЯ
                               ↓
                    
┌─────────────────────────────────────────────────────────────────────┐
│ Стратегия разрешения: "ПОСЛЕДНЕЕ ИЗМЕНЕНИЕ ПОБЕЖДАЕТ"              │
│                                                                     │
│ 1. Сервер сравнивает временные метки обновления                     │
│ 2. Выбирает версию с более поздним updatedAt                        │
│ 3. Применяет изменения в обе стороны                                │
│ 4. Логирует конфликт для проверки                                   │
└─────────────────────────────────────────────────────────────────────┘
                               ↓
                         РЕЗУЛЬТАТ:
┌─────────────────────────────────────────────────────────────────────┐
│ Обе версии содержат:                                                │
│ • status = "Completed" (из таблицы)                                 │
│ • overallComment = "Новый комментарий" (из приложения, т.к. позже) │
└─────────────────────────────────────────────────────────────────────┘

⚠️  РЕКОМЕНДАЦИЯ: Избегайте одновременного редактирования!
```

## Частота синхронизации

```
┌─────────────────────────────────────────────────────────────────────┐
│                      АВТОМАТИЧЕСКАЯ СИНХРОНИЗАЦИЯ                   │
└─────────────────────────────────────────────────────────────────────┘

⏰ Каждые 5 минут (настраивается):

10:00 → Синхронизация
10:05 → Синхронизация
10:10 → Синхронизация
...

Процесс:
1. Сервер проверяет изменения в Supabase KV
2. Сервер проверяет изменения в Google Sheets
3. Применяет изменения в обе стороны
4. Логирует результаты

┌─────────────────────────────────────────────────────────────────────┐
│                       РУЧНАЯ СИНХРОНИЗАЦИЯ                          │
└─────────────────────────────────────────────────────────────────────┘

Администратор может запустить вручную:
• Через кнопку "Синхронизировать сейчас" в интерфейсе
• При добавлении большого количества данных
• При обнаружении расхождений

Рекомендуется при:
✓ Массовом импорте данных
✓ Критически важных обновлениях
✓ Устранении ошибок синхронизации
```

## Что синхронизируется, а что нет

```
✅ СИНХРОНИЗИРУЕТСЯ АВТОМАТИЧЕСКИ:

Из приложения → в таблицу:
• Все текстовые поля (номера, коды, комментарии)
• Числовые данные (влажность, оценки, температуры)
• Статусы и даты
• ПУТИ к фотографиям (path в JSON)
• Булевы значения (isBaseRecipe, isTest)

Из таблицы → в приложение:
• Все текстовые поля
• Числовые данные
• Статусы и даты
• Булевы значения

❌ НЕ СИНХРОНИЗИРУЕТСЯ:

• Сами файлы фотографий (только их пути)
  Фотографии хранятся в Supabase Storage
  В таблице - только ссылки на них

• URL фотографий (генерируются на лету)
  Каждый раз создаются новые подписанные URL
  Для безопасности (истекают через 24 часа)

⚠️ ЧАСТИЧНО СИНХРОНИЗИРУЕТСЯ:

• Погодные данные:
  - Создаются автоматически в приложении
  - Синхронизируются в таблицу
  - Из таблицы не перезаписывают (если уже есть)
```

                         ┌─────────────────┐
                         │   Google Sheets │
                         │  (Таблица Excel)│

                                  │

                         в табличном виде
                                  │
           ┌──────────────────────┼──────────────────────┐
           │                      │                      │
           ↓                      ↓                      ↓
    ┌───────────┐         ┌────────────┐        ┌──────────────┐
    │ Supabase  │         │  Supabase  │        │    Open-     │
    │    KV     │         │   Storage  │        │    Meteo     │

         │                      │                       │

## Процесс загрузки фотографий

```
Оператор загружает фото через приложение:
                    ↓
        ┌───────────────────────┐
        │  1. Файл с те��ефона   │
        │     (photo.jpg)       │
        └───────────┬───────────┘
                    ↓
        ┌───────────────────────┐
        │ 2. Загрузка на сервер │
        │    POST /upload       │
        └───────────┬───────────┘
                    ↓
        ┌───────────────────────┐
        │ 3. Supabase Storage   │
        │  Файл: abc123.jpg     │
        └───────────┬───────────┘
                    ↓
        ┌───────────────────────┐
        │ 4. Возврат пути       │
        │   path: "abc123.jpg"  │
        └───────────┬───────────┘
                    ↓
        ┌───────────────────────┐
        │ 5. Сохранение в цикл  │
        │   recipePhotos: [     │
        │     {path:"abc123.jpg"}│
        │   ]                   │
        └───────────┬───────────┘
                    ↓
        ┌───────────────────────┐
        │ 6. Синхронизация      │
        │   в Google Sheets     │
        │   recipePhotos:       │
        │   [{"path":"abc123"}] │
        └───────────────────────┘

При следующем просмотре:
        ┌───────────────────────┐
        │ 1. Чтение из KV       │
        │   path: "abc123.jpg"  │
        └───────────┬───────────┘
                    ↓
        ┌───────────────────────┐
        │ 2. Генерация signed   │
        │    URL (24ч срок)     │
        └───────────┬───────────┘
                    ↓
        ┌───────────────────────┐
        │ 3. Отображение фото   │
        │   в приложении        │
        └───────────────────────┘
```

## Итоговая схема работы "без рук"

```
┌─────────────────────────────────────────────────────────────────┐
│            ИДЕАЛЬНЫЙ РАБОЧИЙ ПРОЦЕСС                            │
└─────────────────────────────────────────────────────────────────┘

УТРО (09:00):
Вы в Google Sheets:
└→ Создаете 5 новых строк для циклов камер 1,3,5,7,9
└→ Заполняете: номер, код рецепта, тип дерева
└→ Сохраняете таблицу

                    ↓ (через 5 минут)

АВТОСИНХРОНИЗАЦИЯ:
└→ Создает все 5 циклов в приложении
└→ Генерирует UUID для каждого
└→ Обновляет таблицу с новыми ID

                    ↓

Операторы видят в приложении:
└→ 5 новых циклов со статусом "In Progress"
└→ Загружают фото рецептов через телефон
└→ Начинают процесс сушки

                    ↓ (через 5 минут)

АВТОСИНХРОНИЗАЦИЯ:
└→ Обновляет таблицу - добавляет пути к фото

                    ↓ (через 3 дня)

Лидер в приложении:
└→ Открывает завершенный цикл
└→ Добавляет влажность, оценку, фото результата
└→ Статус автоматически → "Completed"

                    ↓ (через 5 минут)

АВТОСИНХРОНИЗАЦИЯ:
└→ Обновляет таблицу со всеми результатами
└→ Вы видите в Google Sheets готовые данные

                    ↓

ВЫ:
└→ Ничего не делаете руками!
└→ Просто смотрите отчеты в таблице
└→ Анализируете статистику
```

Вот и всё! **Полностью автоматизированная система без ручного ввода!** 🎉
