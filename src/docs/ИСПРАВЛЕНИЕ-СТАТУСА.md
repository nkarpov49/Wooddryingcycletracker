# ✅ Исправление определения статуса циклов

## 🔧 Проблема

HTML инструмент показывал **неправильное количество циклов "В процессе"**:
- Показывало: 116 из 130
- На самом деле: 19 из 130

## 🎯 Причина

Инструмент определял статус по полю `status`, но **правильный способ** - проверять наличие `endDate`:

❌ **Старый код (неправильный):**
```javascript
const status = cycle.status || 'Unknown';
const badgeClass = status === 'In Progress' ? 'badge-progress' : 'badge-completed';
```

✅ **Новый код (правильный):**
```javascript
const isCompleted = cycle.endDate && cycle.endDate !== '';
const status = isCompleted ? 'Завершено' : 'В процессе';
const badgeClass = isCompleted ? 'badge-completed' : 'badge-progress';
```

## 📋 Логика определения статуса

### ✅ Цикл ЗАВЕРШЁН если:
```javascript
cycle.endDate && cycle.endDate !== ''
```

**Примеры:**
- ✅ `endDate: "2026-02-23T10:30:00Z"` → **Завершено**
- ✅ `endDate: "2026-01-15T14:20:00Z"` → **Завершено**

### ⚙️ Цикл В ПРОЦЕССЕ если:
```javascript
!cycle.endDate || cycle.endDate === ''
```

**Примеры:**
- ⚙️ `endDate: null` → **В процессе**
- ⚙️ `endDate: undefined` → **В процессе**
- ⚙️ `endDate: ""` → **В процессе**

```javascript
    // ...

```

### 2. Функция `updateStats()`:
```javascript
function updateStats(cycles) {
  const total = cycles.length;
  // Цикл завершён если есть endDate
  const completed = cycles.filter(c => c.endDate && c.endDate !== '').length;
  const inProgress = total - completed;
  
  document.getElementById('totalCount').textContent = total;
  document.getElementById('inProgressCount').textContent = inProgress;
  document.getElementById('completedCount').textContent = completed;
}
```
