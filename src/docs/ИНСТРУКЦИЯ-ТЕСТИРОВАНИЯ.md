# 🧪 Инструкция по тестированию импорта из Google Sheets

## ✅ Всё готово к тестированию!

Система полностью настроена и готова к работе. Следуйте этой инструкции шаг за шагом.

---

## 📋 Что вам понадобится

1. **Google Sheets** - ваша таблица
2. **Apps Script** - скрипт для отправки данных
3. **HTML инструмент** - для проверки результата (`/docs/check-cycles-database.html`)
4. **5-10 минут времени**

---

## 🎯 Шаг 1: Найдите тестовый цикл

1. Откройте в браузере: `/docs/check-cycles-database.html`

2. Найдите любой цикл со статусом **"В процессе ⚙️"**

3. Запишите его данные:
   ```
   Камера: 18
   Порядковый номер: 9954
   Порода: Birch285
   UUID: abc-123-def-456...
   ```

**Пример того что вы увидите:**
```
═══════════════════════════════════════════════
Камера 18 • №9954 | В процессе ⚙️
═══════════════════════════════════════════════
UUID: abc-123-def-456-789-012
Начало: 20.02.2026, 08:00:00
Окончание: -
Порода: Birch285
Влажность: - %
Рейтинг: - ⭐
```

## 📊 Шаг 2: Подготовьте Google Sheets

### 2.1 Создайте структуру таблицы

**Строка 1 (заголовки):**
```
| A      | B          | C      | D           | E      | F           |
| Камера | Порода     | Старый | Дата оконч. | Новый  | Дата начала |
```

### 2.2 Добавьте тестовую строку

**Строка 2 (данные):**
```
| 18 | Beržas235 | 9954 | 2026-02-23 | 9955 | 2026-02-23 |
```

**⚠️ Важно:**
- **Камера (A):** Номер вашего тестового цикла (18)
- **Порода (B):** Литовское название для НОВОГО цикла (Beržas235)
- **Старый № (C):** Номер из HTML инструмента (9954)
- **Дата оконч. (D):** Сегодняшняя дата
- **Новый № (E):** Старый + 1 (9955)
- **Дата начала (F):** Та же что и дата окончания

**Допустимые форматы даты:**
- `2026-02-23` (YYYY-MM-DD)
- `23.02.2026` (DD.MM.YYYY)
- `2026-02-23 10:00:00` (с временем)

## 🔧 Шаг 3: Обновите Apps Script

1. В Google Sheets откройте: **Расширения → Apps Script**

2. **Удалите весь существующий код** в редакторе

3. **Откройте файл** `/docs/google-apps-script.gs` в вашем редакторе кода

4. **Скопируйте весь код** (Ctrl+A, Ctrl+C)

5. **Вставьте в Apps Script** (Ctrl+V)

6. **Сохраните** (Ctrl+S или иконка дискеты)

7. **Проверьте константы** в начале файла:
   ```javascript
   const URL_WEBHOOK = 'https://zzteuolgibfbulqdavof...';
   const AUTH_TOKEN = 'eyJhbGciOiJIUzI1NiIsI...';
   const SHEET_NAME = 'Лист1'; // ← Измените если ваш лист называется иначе
   ```

## ✔️ Шаг 4: Проверьте настройки

В Apps Script выполните функцию:
```javascript
testConfiguration()
```

**Как выполнить:**

## 🚀 Шаг 5: Запустите синхронизацию

В Apps Script выполните функцию:
```javascript
manualSync()
```

**Ожидаемый результат:**
```
=== РУЧНАЯ СИНХРОНИЗАЦИЯ ===
Последняя строка: 2

=== ОБРАБОТКА СТРОКИ 2 ===
Данные строки для отправки:
{
  "rowNumber": 2,
  "chamberNumber": 18,
  "woodTypeLithuanian": "Beržas235",
  "oldSequentialNumber": "9954",
  "oldCycleEndDate": "2026-02-23T10:00:00Z",
  "newSequentialNumber": "9955",
  "newCycleStartDate": "2026-02-23T10:00:00Z"
}

Отправка запроса на https://zzteuolgibfbulqdavof...
Ответ сервера (HTTP 200):
{
  "success": true,
  "message": "Обработка строки запущена",
  "rowNumber": 2,
  "estimatedTime": "2-3 минуты"
}

✅ Успех! Строка 2 отправлена на обработку
Ответ сервера: Обработка строки запущена
Ожидаемое время обработки: 2-3 минуты
```

**⏳ Важно:** Сервер обрабатывает запрос **асинхронно**:
1. Закрывает старый цикл (0 сек)
2. Ждёт 2.5 минуты
3. Создаёт новый цикл (2.5 мин)

**Общее время: ~3 минуты**

```javascript
resetRowStatus(2)  // 2 = номер строки в таблице
```

**Результат:**
```
Сброс статуса строки 2...
Отправка запроса на https://...

{"success":true,"message":"Статус строки 2 сброшен..."}
✅ Статус строки 2 сброшен! Теперь можно запустить manualSync() снова.
```

Если хотите очистить тестовые данные:

4. Создайте новый цикл 18/9954 вручную

```javascript
manualSync()
```

## ❌ Устранение неполадок

### Проблема 1: "Строка уже обработана"

**Причина:** Строка была обработана ранее.

**Решение:**
```javascript
resetRowStatus(2)  // Сбросить статус
manualSync()       // Повторить синхронизацию
```

---

### Проблема 2: Ошибка HTTP 400

**Пример логов:**
```
❌ Ошибка HTTP 400
{
  "error": "Отсутствуют обязательные поля",
  "required": ["rowNumber", "chamberNumber", ...]
}
```

**Причина:** Не все поля заполнены в таблице.

**Решение:**
1. Проверьте что ВСЕ 6 столбцов (A-F) заполнены
2. Проверьте форматы данных:
   - Камера: число 1-21
   - Порода: текст (например: Beržas235)
   - Номера: текст, 4 цифры (например: 9954)
   - Даты: корректный формат даты

---

### Проблема 3: Цикл не найден

**Пример логов:**
```
Ответ сервера (HTTP 200):
{"success":true,...}

✅ Успех!
```

**Но в HTML инструменте ничего не изменилось.**

**Причина:** Старый цикл с таким номером не существует.

**Решение:**
1. Проверьте номер камеры и порядковый номер
2. Убедитесь что цикл со статусом "В процессе" существует
3. Проверьте правильность данных в столбцах A и C

---

### Проблема 4: Порода не конвертировалась

**Пример:** Новый цикл имеет породу "" (пусто).

**Причина:** Литовское название не поддерживается.

**Решение:**
1. Проверьте список поддерживаемых пород в документации
2. Добавьте новую породу в `/supabase/functions/server/wood-type-mapping.ts`

**Поддерживаемые породы:**
- Beržas235, Berzas235 → Birch235
- Beržas285, Berzas285 → Birch285
- Alksnis235 → Alder235
- Ąžuolas235, Azuolas235 → Oak235
- Uosis235 → Ash235
- Klevas235 → Maple235
- Skroblas235 → Scroblas235

---

### Проблема 5: Слишком долго ждать

**Причина:** Задержка 2.5 минуты может показаться долгой.

**Решение (для разработки):**

Измените задержку в `/supabase/functions/server/google-sheets-sync.ts`:
```typescript
// Было:
const delayMs = 2.5 * 60 * 1000; // 2.5 минуты

// Станет:
const delayMs = 10 * 1000; // 10 секунд (только для тестирования!)
```

**⚠️ Внимание:** Не используйте короткую задержку в продакшене!
