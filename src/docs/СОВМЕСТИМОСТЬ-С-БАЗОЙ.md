## ✅ Что делает система при закрытии старого цикла

### Код (абсолютно безопасный):

```typescript
// Ищем цикл по sequentialNumber
const cycles = await kv.getByPrefix("cycle_");
const oldCycle = cycles.find((c: any) => 
  c.sequentialNumber === row.oldSequentialNumber &&
  c.chamberNumber === row.chamberNumber
);

// КРИТИЧЕСКИ ВАЖНО: Используем spread operator
const updatedCycle = {
  ...oldCycle,  // ← Копируем ВСЕ существующие поля
  status: 'Completed',  // ← Меняем только статус
  endDate: row.oldCycleEndDate,  // ← И дату окончания
};

// Сохраняем обратно
await kv.set(`cycle_${oldCycle.id}`, updatedCycle);
```

### Что это значит:

1. **Читаем** карточку из KV store
2. **Копируем** ВСЕ её поля через `...oldCycle`
3. **Перезаписываем** только 2 поля: `status` и `endDate`
4. **Сохраняем** обратно в KV store с тем же ключом

## 🔐 Гарантии безопасности

### 1. Spread operator (`...oldCycle`)

Этот оператор **копирует ВСЕ поля** из объекта, включая:
- Известные нам поля
- Неизвестные нам поля
- Legacy поля
- Будущие поля

**Даже если мы не знаем о поле - оно сохранится!**

### 2. Только 2 поля меняются

При закрытии цикла **ГАРАНТИРОВАННО** меняются только:
- `status` → `'Completed'`
- `endDate` → дата из Google Sheets

**Всё остальное - копия!**

### 3. Новая карточка - новая запись

Новый цикл **ВСЕГДА** создаётся как новая строка в KV таблице.

**Никогда не перезаписывает существующие!**

### 4. UUID неизменен

UUID старой карточки **НИКОГДА** не меняется.

Ключ в KV таблице остаётся тем же: `cycle_<тот же UUID>`
