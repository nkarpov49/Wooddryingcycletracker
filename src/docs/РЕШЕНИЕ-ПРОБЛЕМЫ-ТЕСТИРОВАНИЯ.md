# 🔧 Решение: "Строка уже обработана"

## Проблема

При запуске `manualSync()` вы получили:
```
{"success":true,"message":"Строка уже обработана","skipped":true}
```

Это значит, что система **уже обработала** эту строку ранее и пропустила её, чтобы не создавать дубликаты.

---

## ✅ Решение 1: Сброс статуса строки (для повторного тестирования)

### Шаг 1: Обновите код в Apps Script

1. Откройте Apps Script (Расширения → Apps Script)
2. Найдите код в редакторе
3. **Замените весь код** новым из файла `/docs/google-apps-script.gs`
   - Новая версия включает функцию `resetRowStatus()`
4. Сохраните (Ctrl+S или Cmd+S)

### Шаг 2: Сбросьте статус строки

1. В Apps Script выберите функцию **`resetRowStatus`** в dropdown меню
2. Нажмите **▶️ Выполнить**
3. Проверьте логи: **Вид** → **Журналы**

Вы должны увидеть:
```
Сброс статуса строки 2...
Ответ сервера (код 200):
{"success":true,"message":"Статус строки 2 сброшен. Теперь её можно обработать снова."}
✅ Статус строки 2 сброшен! Теперь можно запустить manualSync() снова.
```

### Шаг 3: Повторите тест

1. Выберите функцию **`manualSync`**
2. Нажмите **▶️ Выполнить**
3. Теперь строка будет обработана!

## ✅ Решение 2: Добавить новую тестовую строку

Если не хотите сбрасывать статус, просто добавьте новую строку:

### Вариант А: С другим номером цикла

```
18 | 9955 | 9960 | Beržas285 | 2026-02-23 19:00:00 | 2026-02-23 19:05:00
```

### Вариант Б: С другой камерой

```
17 | 9950 | 9951 | Klevas235 | 2026-02-23 19:00:00 | 2026-02-23 19:05:00
```

**Важно:** Используйте номера циклов, которые **существуют в вашей базе**, иначе старый цикл не закроется.

## 📋 Правильная последовательность для теста

### Шаг 1: Найдите существующий активный цикл

В приложении найдите цикл со статусом **"In Progress"** (В процессе).

**Пример:**
- Камера: 18
- Порядковый номер: 9954
- Статус: In Progress

### Шаг 2: Придумайте новый номер

Новый номер должен быть **больше** старого и **не существовать** в базе.

**Пример:**
- Старый: 9954
- Новый: 9959 ✅ (не существует в базе)

### Шаг 3: Добавьте строку в Google Sheets

| A | B | C | D | E | F |
|---|---|---|---|---|---|
| 18 | 9954 | 9959 | Beržas285 | 2026-02-23 19:00:00 | 2026-02-23 19:05:00 |

### Шаг 4: Запустите синхронизацию

Выберите ОДИН из способов:

**Способ А: Сброс + повторная обработка (если строка уже есть)**
1. `resetRowStatus()` - сбросить статус
2. `manualSync()` - обработать снова

**Способ Б: Новая строка (если добавили новую)**
1. `manualSync()` - обработать новую строку

### Шаг 5: Подождите и проверьте

**Через 10-15 секунд:**
- Откройте приложение
- Найдите цикл 9954

- Проверьте: **Дата окончания = 2026-02-23 19:00** ✅

**Через 3 минуты:**
- Обновите приложение
- Должен появиться цикл 9959
- Проверьте: **Статус = "In Progress"** ✅

## ⚠️ Важные замечания

### 1. Номер старого цикла должен совпадать ТОЧНО

Если в базе `"9954"`, в таблице должно быть `9954` (не `09954`, не `"9954"` с кавычками).

### 2. Используйте 4-значные номера

Хотя в вашем случае номер 9954 (уже 4 цифры), если будет номер 123, в таблице нужно писать `0123`.

### 3. Литовское название породы

Используйте **ТОЧНО** такие названия:
- `Beržas235` или `Berzas235` (без диакритики)
- `Beržas285` или `Berzas285`
- `Klevas235`
- `Alksnis235`
- `Ąžuolas235` или `Azuolas235`
- `Uosis235`
- `Skroblas235`

### 4. Формат дат

```
ГГГГ-ММ-ДД ЧЧ:ММ:СС
2026-02-23 19:00:00
```

Или используйте формат даты Google Sheets (будет автоматически конвертирован).

## 🆘 Если всё равно не работает

### Проверка 1: Логи Apps Script

**Вид → Журналы выполнения** (View → Execution log)

Что искать:
- ❌ Ошибки HTTP (401, 404, 500)
- ✅ "Ответ сервера (код 200)"
- ✅ "✅ Успех! Строка X отправлена"

### Проверка 2: Логи сервера

Откройте в браузере:
```
GET https://zzteuolgibfbulqdavof.supabase.co/functions/v1/make-server-c5bcdb1f/sheets/sync-logs?limit=10
Authorization: Bearer <ваш anon key>
```

Что искать:
```json
{
  "logs": [
    {
      "timestamp": "2026-02-23T18:58:20Z",
      "action": "complete_old",
      "success": true,
      "message": "Цикл 9954 завершён"
    },
    {
      "timestamp": "2026-02-23T19:01:20Z",
      "action": "create_new",
      "success": true,
      "message": "Новый цикл 9959 создан"
    }
  ]
}
```

### Проверка 3: Статус строки

Откройте в браузере:
```
GET https://zzteuolgibfbulqdavof.supabase.co/functions/v1/make-server-c5bcdb1f/sheets/row-status/2
Authorization: Bearer <ваш anon key>
```

Если вернёт `{"processed": true}` - строка обработана.  
Если вернёт `{"processed": false}` - строка НЕ обработана.

### Проверка 4: Существует ли старый цикл

В приложении проверьте:
- Есть ли цикл с номером 9954?
- У него камера 18?
- У него статус "In Progress"?

Если хотя бы один ответ "нет" - система не найдёт цикл для закрытия.
